Hexo + GitHub Actions 实现博文自动部属

Long & Kelly
2023-04-06
DevOps
Hexo

架构图

操作步骤

准备 Deploy key

要准备 Deploy key 就要用 ssh-keygen 生成公钥私钥

ssh-keygen -t ed25519 -C "<YOUR_EMAIL>" -f id_rsa

可以替换成你的 GitHub Email 或者照下面填写 GitHub Action Bot 的 Email。

Enter passphrase 就直接按下 Enter 跳过。
之后就会生成 id_rsa.pub 跟 id_rsa 两个文件。

  • id_rsa.pub 是公钥
  • id_rsa 是私钥

GitHub Pages 专案中放上公钥

点到要部属的 GitHub Pages 的 Repo
Setting -> Deploy keys -> Add deploy key

将 deploy.pub 公钥内容贴到 Key 这里。

Allow write access 打勾，这样才能给 GitHub Actions 部属。

Hexo Source 专案中放上私钥

点到放 Hexo Source 的 Repo

Setting -> Secrets -> Actions -> New repository secret

  • Name：DEPLOY_KEY
  • Value：将 deploy 私钥内容贴到这里。

设定 GitHub Actions

点到放 Hexo Source 的 Repo
按下 Actions 里面 set up a workflow yourself

创建 .github/workflows/deploy-pages.yml

特别注意

  • yml 里 DEPLOY_KEY 要和私钥部分一致，否则无法部署会 GitHub Page 仓库
  • yml 里 Clone Themes 要和代码内使用的主题一致，否则生成的文件将会全部是空文件
