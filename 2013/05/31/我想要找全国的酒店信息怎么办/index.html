我想要找全国的酒店信息怎么办？

Long & Kelly
2013-06-01
Security
Security

今日需要获取一些数据，这些数据从网上找不到下载的地方。这里以酒店数据为例。下同，今日想获取全国的酒店 LBS 信息的数据，包括但不局限于酒店名称，酒店经纬度，酒店入住价格等信息。

于是想到可以分析现有 APP 接口得到数据，通常来讲，APP 接口的返回的数据是 json 和 xml 格式的。
所需工具：抓包工具 Wireshark 之类的，代理工具(用于在手机或者是 Pad 上使用)，为了方遍，直接采用 Fiddler2 工具，自带有代理选项
手机客户端或者是 Pad 客户端上设置好代理，这个不多说。
打开 APP，刷几个列表，发现 Fiddle 里面就有很多数据了，包括本身这个 app 的，还有这个 app 的统计信息，一般的没战略的 app 可能会没有统计信息。

好吧。。。在 fiddle 里面找到一个请求，是 http 的，post，有 host，有 param，本地构造。

<html>
    <form method="post" action="http://host:port/Finder.php?type=city_hot_zones&city_id=1&client_version=1.6.4&client_agent=iPad&market=AppStore">
        <input name="type" value="city_hot_zones">
        <input name="city_id" value="1">
        <input name="client_version" value="1.6.4">
        <input name="client_agent" value="iPad">
        <input name="market" value="AppStore">
        <input type="submit">
    </form>
</html>

ok，然后浏览器访问，发现返回是 xml 的数组，包含名字和这个酒店的经纬度。

于是乎，如果可以构造请求，那么这个 app 上面的所有的地理位置数据理论上讲是都可以下载了。httpclient 模拟请求 over，然后等数据。。。

反思 中间的数据是不是应该采用加密措施。

不要说就我想到这么搞，其实他们的数据也是这么搞到的，说不定哈。。。
分析一个 APP—从功能上，从界面设计上，从盈利模式上，从市场推广上，现在到了，从数据交换的协议上。。。
